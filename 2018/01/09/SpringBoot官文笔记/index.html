关于Application的注解：@SpringBootApplication包含@EnableAutoConfiguration与@ComponentScan及@Configuration。
也就是说下面两种方式是等价的:

@Configuration可以配置给多个类，@SpringBootApplication或 @EnableAutoConfiguration仅给一个扮演Application的类
与@Configuration配合，你还可以使用@Import (class)，@ImportResource (xml)
既然SpringBoot会自动配置，那么我们如何查看其自动配置过程呢？使用--debug参数启动即可

禁用某些类的自动配置行为：
@EnableAutoConfiguration(exclude={DataSourceAutoConfiguration.class})
如果类不再路径中
@EnableAutoConfiguration(excludeName={“DataSourceAutoConfiguration.class”})
当然你也可以通过配置文件直接排除：
spring.autoconfigure.exclude=xxx.xxxx.DataSourceAutoConfiguration

热部署/Hot swapping

1、JRebel（收费）
2、Spring Loaded
3、spring-boot-devtools

使用Spring Loaded:

当系统通过 mvn spring-boot:run启动或者 右键application debug 启动java文件时，系统会监视classes文件，当有classes文件被改动时，系统会重新加载类文件，不用重启启动服务
ps：IDEA下需要重新编译文件，按ctl+shift+f9后才能生效。eclipse下则自动编译生效。

使用Spring-boot-devtools:

注意：该依赖会在运行fully packaged application时(即使用java -jar或使用特殊的classloader运行时 )被自动禁用.
repackaged archives默认不会包含devtools依赖. 如果你想包含它，你需要配置excludeDevtools build property,如下

使用devtools的好处：
1、在开发时禁用模版引擎缓存，禁用SpringMVC静态资源响应缓存。而不需要你手动配置application.properties文件
参见devtools改变的属性列表
2、类改变自动重启
3、具体路径规则什么的，见官文
4、支持浏览器livereload插件配合。
关闭devtools自动重启：spring.devtools.restart.enabled=false

SpringApplication类

关于Application event与监听器：https://docs.spring.io/spring-boot/docs/1.5.9.RELEASE/reference/htmlsingle/#boot-features-application-events-and-listeners

Web environment:
SpringApplication创建的ApplicationContext有两种情况： AnnotationConfigApplicationContext or AnnotationConfigEmbeddedWebApplicationContext 区别是后者用于web环境，怎么选择，spring通过判断classpath中是否存在一些类。有个时候即使存在，你也不想使用web环境，那么你可以显式配置：SpringApplication.setWebEnvironment(false).

访问命令行参数：
通过pringApplication.run(…,args)传入的命令行参数，可以通过注入org.springframework.boot.ApplicationArguments来获取

同时由于Springboot会将CommandLinePropertySource注册到Environment,你甚至可以直接通过@Value注入。(具体用法没例子)

ApplicationRunner or CommandLineRunner：
用于在SpringApplication启动后，但SpringApplication.run(…​)执行完成之前。两者都可以获取命令行参数，前者通过String[],后者通过ApplicationArguments

如果他们有多个，而且要按顺序执行，那么你可以实现org.springframework.core.Ordered接口或者直接使用 @org.springframework.core.annotation.Order

Application exit:
SpringApplication会注册jvm shutdown hook来实现优雅关闭。退出时会进行Spring lifecycle callbacks，如调用DisposableBean接口实现，@PreDestroy注解的类。你甚至可以自定义exitcode。

通过spring.application.admin.enabled属性，你还可以开放SpringApplicationAdminMXBean来远程管理应用。

支持：properties，yaml，环境变量，命令行参数。这些属性值可以直接通过@Value注入。

日志

SpringBoot内部日志使用Commons Logging.对外支持Log4j2,logback.默认使用Logback

debug日志：

还有个trace与debug类似。

日志文件输出:
在application.properties指定logging.file(指定文件名，如my.log,logs/my.log)或logging.path(指定目录，如/var/log)属性
默认日志文件会以10MB为单位rotate，同时日志级别为INFO.
指定日志级别：
logging.level.root=WARN
logging.level.org.springframework.web=DEBUG
logging.level.org.hibernate=ERROR

日志配置文件命名：(springboot会在classpath根目录下自动检测配置文件，你也可通过logging.config指定位置)
logback： logback-spring.xml(推荐，可以使用特殊用法，也便于Spring控制),logback.xml
log4j2: log4j2-spring.xml(推荐)或log4j2.xml

如果classpath中存在多个实现，你也有不同实现的配置文件，这时候，如果你想仅使用一个，可通过配置org.springframework.boot.logging.LoggingSystem=LoggingSystem的全限定类名或者none来指定，不过一般不需要这么做。

SpringMVC的自动配置：

  • 包含ContentNegotiatingViewResolver and BeanNameViewResolver.
  • 支持static resources及 WebJars .
  • 自动注册Converter, GenericConverter, Formatter beans
  • 支持HttpMessageConverters.
  • 自动注册MessageCodesResolver.
  • Static index.html support.
  • Custom Favicon support .
  • Automatic use of a ConfigurableWebBindingInitializer bean.

如果你想保持SpringBoot为MVC提供的这些特性，并向添加额外的配置，如(interceptors, formatters, view controllers etc.) ,那么你可以继承WebMvcConfigurerAdapter并使用@Cofiguration注解类。但是千万不要使用@EnableWebMvc(除非你想完全自己来配置SpringMVC)

配置HttpMessageConverters:
SpringMVC使用它来convert http请求和响应。默认配置能够很好地工作，如自动json、xml转换，utf-8编码。你仍然可以自定义，比如使用fastjson。

所有的HttpMessageConverters bean都会被添加到converters list中。所以此处命名无所谓。
Custom JSON Serializers and Deserializers：见官文。

静态资源：
classpath下的/static or /public or /resources or /META-INF/resources
它使用ResourceHttpRequestHandler 来处理，所以在WebMvcConfigurerAdapter中，你可以通过addResourceHandlers方法来配置。

ConfigurableWebBindingInitializer：
Spring MVC uses a WebBindingInitializer to initialize a WebDataBinder for a particular request. If you create your own ConfigurableWebBindingInitializer @Bean, Spring Boot will automatically configure Spring MVC to use it.

模版引擎：

Security

如果Spring Security 在classpath中，那么将会对所有的HTTP endpoints启动basic认证.
默认用户名：user,密码，启动时看到这一行 Using default security password: 78fa095d-3f4c-48b1-ad50-e24c31d5cf35
如果看不到，请将org.springframework.boot.autoconfigure.security日志级别设成INFO
如果你想指定密码，通过配置：security.user.password
完整的配置项，由SecurityProperties类定义

默认security自动配置：
SecurityAutoConfiguration
SpringBootWebSecurityConfiguration 给web security
AuthenticationManagerConfiguration 给非web程序
@EnableWebSecurity禁用antuhentication manager配置，但除了Actuator’s security.

通过继承WebSecurityConfigurerAdapter来自定义。
自定义AuthenticationManager注册为bean
详见官方文档

Web应用可以享受到的SpringBoot提供给Security的特性：

  • An AuthenticationManager bean with in-memory store and a single user (see SecurityProperties.User for the properties of the user).
  • Ignored (insecure) paths for common static resource locations (/css/ , /js/ , /images/ , /webjars/ and /favicon.ico).
  • HTTP Basic security for all other endpoints.
  • Security events published to Spring’s ApplicationEventPublisher (successful and unsuccessful authentication and access denied).
  • Common low-level features (HSTS, XSS, CSRF, caching) provided by Spring Security are on by default.

hazelcast

Hazelcast是一个高度可扩展的数据分发和集群平台。特性包括：

  • 提供java.util.{Queue, Set, List, Map}分布式实现。
  • 提供java.util.concurrency.locks.Lock分布式实现。
  • 提供java.util.concurrent.ExecutorService分布式实 现。
  • 提供用于一对多关系的分布式MultiMap。
  • 提供用于发布/订阅的分布式Topic（主题）。
  • 通过JCA与J2EE容器集成和事务支持。
  • 提供用于安全集群的Socket层加密。
  • 支持同步和异步持久化。
  • 为Hibernate提供二级缓存Provider 。
  • 通过JMX监控和管理集群。
  • 支持动态HTTP Session集群。
  • 利用备份实现动态分割。
  • 支持动态故障恢复。
    特性说明：http://hao.jobbole.com/hazelcast/
